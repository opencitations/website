The Crossref Open Citation Identifier (COCI)

The COCI project takes in consideration a large number of bibliographic resources made available and retrieved from Crossref, and annotates/elaborates their citing information.

The COCI dataset contains a total of:
[N] citing bibliographic resources.
[N] cited resources.

A COCI resource is therefore an entity which defines a citation between two different bibliographic resources. The COCI textual format is structured with: the lower-case letters “ci” followed by a colon, followed by two numbers separated by a dash. The first number identifies the citing resource, while the second one identifies the cited resource. The Crossref bibliographic resource identifiers have a supplier prefix equl to "020". For example, ci/02001010806360107050663080702026306630509-02001010806360107050663080702026305630301 is a valid COCI.

The COCI data dumps are available in CSV, NT, and JNL formats. These dumps are stored in Figshare and open for external usage.

You can work and iteract with the COCI triplestore using the following services/applications:

  • A triplestore searching interface (built using OSCAR)

  • A resource browser (built using LUCINDA). In order to use this service, a resource identifier must be specified in the URL address.
    For instance: /index/coci/browser/ci/02001010806360107050663080702026306630509-02001010...

  • A SPARQL editor to query the COCI triplestore

  • A REST API service
